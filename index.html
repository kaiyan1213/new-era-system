          Auto Commission — from 收单追踪 (Installed orders)
          

            
            
            
          

        

        

          Pulls every new order from this month's Coway payslips (the authoritative list, across all proxies) and cross-checks it against 收单追踪's claims, matched purely by customer name (any "(nickname)" or other notes staff add after the name are ignored) — via the "收单追踪 Staff Name" set on each staff member on the Payroll tab. Only orders verified against an actual payslip count toward commission (RM50/order for orders 1–20, +RM1000 bonus at the 10th, +RM1000 at the 20th, RM100/order from the 21st onward). Payslip orders with no matching staff claim show up below for manual assignment.
        

        

          
Bulk Manual Match

          
Paste a list of customer names (one per line) and pick a staff member — the system will search this month's Coway payslips for each name and assign any match to that staff. Names that don't match any payslip order for this month are skipped and reported back. All matched names in one batch get the same commission algorithm — use Outsider here if this whole batch is non-ad-sourced customers, otherwise leave it on the staff's default.

          Acc Below (auto) + Adjustments (manual)
          
        

        

          Acc Below = the money this manager's proxy accounts didn't collect this period — auto-computed as the sum of each of their proxies' (Coway payslip − New Era invoice) for the period, the same PIC liability shown on the Directory page, kept with the manager instead of collected from the proxy directly. It's a deduction owed back to New Era, not a bonus — see the Overview tab for how it nets against Com/Claim/Mistake. Adjustments (Claim/Mistake/Gift/Loan Claim/etc) are a separate, manually-entered category: positive = manager receives, negative = manager owes. On the final settlement (Overview tab, Payslip/Calculation documents), Mistake stays its own separate figure while every other sub-category here (Claim/Gift/Loan Claim/Alpha C Claim/Return Back/Other) rolls into one "Claim" figure — this table is just for tracking the detailed reasons. Applies to all 5 managers including Zehao, who has Acc Below and Adjustments but no Manager Comm pool share.
